# Heikin Ashi Monitoring Service

A serverless service that ingests OHLC quotes from Yahoo Finance, computes
Heikin Ashi candles, detects pattern signals (color change, strong candle,
doji), and dispatches rich email alerts enriched with a fundamental analysis
note generated by Claude on AWS Bedrock and an inline JFreeChart-rendered
HA chart.

The full behavioral specification lives in [`CLAUDE.md`](CLAUDE.md). That
document is the source of truth for architecture, data model, error
catalog, and per-block Gherkin scenarios.

## Stack

| Layer            | Choice                                                   |
|------------------|----------------------------------------------------------|
| Language         | Java 25 (Amazon Corretto)                                |
| Framework        | Micronaut 4.10.x                                         |
| Build            | Maven                                                    |
| Compute          | AWS Lambda (JVM + SnapStart, arm64)                      |
| Database         | DynamoDB single-table, on-demand                         |
| Market data      | Yahoo Finance (`de.sfuhrm:YahooFinanceAPI`)              |
| AI               | AWS Bedrock + Claude (Converse API, manual tool loop)    |
| Charting         | JFreeChart (headless)                                    |
| Email            | Apache Commons Email + SES v2 raw send                   |
| IaC              | Terraform                                                |
| CI/CD            | GitHub Actions (OIDC to AWS)                             |

The Java tree follows hexagonal layering as defined in CLAUDE.md
§13: `domain` → `application` → `infrastructure` → `orchestration`.
The `domain` layer must not import any AWS SDK or third-party
infrastructure type.

## Build

Requires JDK 25 and Maven 3.9+.

```bash
mvn verify           # compile + run tests + coverage gate
mvn package          # build the Lambda jar (Shade)
mvn -pl . test       # unit + integration tests
```

## Deploy

Deployment is automated via GitHub Actions on push to `main` (OIDC
federated, no static keys). See CLAUDE.md §12 for the pre-deploy
checklist (SES production access, Bedrock model access, SSM
SecureString parameters, etc.).

The Terraform stack is split into:

- a **bootstrap** stack (state bucket, lock table, OIDC provider, deploy
  role) — applied once, manually;
- the **main** stack (DynamoDB, Lambdas, EventBridge schedules, SNS DLQ,
  IAM, alarms) — applied by CI.

## Operations

A small CLI script wraps the AWS SDK to manage instruments without an
HTTP API. Examples (see CLAUDE.md §10):

```bash
mon register --ticker AAPL --exchange NASDAQ
mon config set --id <uuid> --policy ROLLING_WINDOW --window 200
mon config recipients --id <uuid> --add me@example.com
mon run --instruments <uuid1>,<uuid2>
```
